# know-the-power
"Know The Power" -  Відстеження тренувань, вправ та прогресу!

Головна логіка
1. Авторизація та локальне сховище
Користувач починає з ідентифікації через логін та пароль, після чого система підключається до локального файлу бази даних. Всі персональні метрики — стать, вік, зріст та початкова вага — зберігаються у цьому файлі назавжди. Це дозволяє додатку при кожному новому запуску автоматично зчитувати профіль та завантажувати персональні дані без повторного введення.
2. Архівація та календарна прив’язка
Головне меню працює як точка доступу до історичних даних. Кожен запис про тренування або їжу автоматично отримує мітку часу. Коли ти обираєш дату в календарі, наприклад 13.04.2026, програма виконує пошуковий запит до файлу сховища і відтворює стан дня: які вправи були виконані та скільки калорій спожито саме тоді. Це забезпечує безперервність твого щоденника протягом років.
3. Запис тренувань та харчування
Модуль активності фіксує дані про кругові чи силові тренування безпосередньо у базу. Кожна додана вправа, кількість повторів та підходів записується як окрема подія, прив’язана до конкретного ідентифікатора дня. Аналогічно працює і модуль харчування: назва продукту та його об’єм зберігаються у списку за конкретну дату. Навіть якщо програма вимкнеться, дані залишаться у безпеці у файлі записів.
4. Динаміка ваги та щоденних метрик
Особлива увага приділяється зберіганню щоденних замірів ваги та кроків. Замість того, щоб просто оновлювати одне число, програма створює хронологічну стрічку записів. Це дозволяє системі при кожному зважуванні порівнювати поточне значення з попередніми періодами. Наприклад, додаток бачить, що сьогодні твоя вага 96 кг, зчитує з файлу запис за минулий тиждень (де було 100 кг) і автоматично розраховує прогрес.
5. Аналітика на основі збереженої історії
Статистика — це результат обробки всього масиву накопичених даних. Коли ти відкриваєш вікно звітів, програма «прокручує» файл сховища за останній тиждень або місяць. Вона сумує калорії, рахує кількість тренувань та будує лінію прогресу в силових вправах. Завдяки тому, що кожен крок було зафіксовано у файлі, ти отримуєш порівняння «було — стало».

Сутності:
1. User (Користувач) — головний об'єкт. Містить у собі дані для входу (email, хеш пароля) та фізичний профіль: стать, вік, зріст, початкова вага. 
2. Workout (Тренування) — запис про спортивну активність. Кожне тренування має унікальний ID, дату, назву та тип (силове чи кругове). Воно є батьківським об'єктом для вправ.
3. ExerciseLog (Запис вправи) — деталізація всередині тренування. Тут записуємо назву вправи, кількість підходів, повторень, вагу обтяження та час відпочинку. Кожен такий запис прив'язаний до конкретного Workout.
4. FoodEntry (Запис харчування) — одиниця щоденника їжі. Містить назву продукту, кількість (грам/мл/шт) та час споживання. Кожен запис має прив'язку до дати.
5. DailyMetric (Щоденні показники) — окрема сутність для відстеження фізичних змін. Сюди записується вага користувача та кількість кроків за конкретне число. Саме з цих записів програма буде будувати графіки схуднення чи активності.

Ендпоінти:
POST /api/auth/register — Реєстрація 
POST /api/auth/login — Вхід та отримання токена. 
GET /api/journal/{date} — Повний зріз дня (метрики + їжа + тренування). 
PATCH /api/stats/metrics — Оновлення ваги та кроків на конкретну дату. 
POST/PATCH/DELETE /api/food — Повний цикл керування харчуванням. 
POST/PATCH/DELETE /api/workouts — Керування тренуваннями. 
POST /api/workouts/{id}/exercise — Додавання вправ до плану. 
GET /api/stats/weekly — Зведена аналітика прогресу.
